# Texture ID to File Mapping Configuration
# This file defines the relationship between texture IDs and texture files
# 
# Format: ID:filename:display_name
#   ID: Integer from 0-255 (0 is always empty/air)
#   filename: Name of the .h file in include/data/textures/ (or "null" for empty)
#   display_name: Human-readable name for display in tools
#
# Lines starting with # are comments and will be ignored
# Empty lines are also ignored
#
# IMPORTANT: 
# - ID 0 is reserved for empty space (no wall)
# - Maximum supported IDs: 0-255 
# - When adding new textures, assign the next available ID
# - Both map_editor.py and C++ code read this file

# Empty space (required)
0:null:Empty
